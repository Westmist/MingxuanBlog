JDBC连接数据库

数据库
发布日期:   2019-10-20
作者:   明轩
阅读次数:  

导入jdbc依赖包

导入JDBC驱动包mysql-conector-java-version.jar

加载mysql驱动

使用Class.foranme("com.mysql.cj.jdbc.Driver")加载驱动(mysql8以上版本)

在低版本的驱动加载则是Class.forname("com.mysql.jdbc.Driver")

使用驱动类的方法连接数据库

使用驱动的getConection()方法实例化java.sql.Conection类对象

URL格式为jdbc:mysql://localhost:3306/database_name

java.sql.Conection con = Driver.getConection("URL","USER","PASSWORD")

Tip:可封装一个自定义的JDBC连接类，实现代码的复用，只需在该类的方法中返回一个java.sql.Conection类对象

对数据库进行操作 

//使用conection对象创建一个Statement  
Statement sta = conection.createStatement()  
//执行sql语句并接受结果
java.sql.ResultSet res = sta.execute(String sql)
//res依次接受数据库的每一行，使用res的`getType(int index)`方法获取具体的某一项值  
while(res.next()){
    //遍历打印表中每一行的第二项值，索引从1开始
    System.out.println(res.getString(2));
}

动态sql语句

动态查询语句PreparedStatement,使用该类带替使用Statement类，可防止sql注入，只有在查询时才知道具体的查询内容

//使用'?'代替要查询的内容
String sql = "SELECT * FROM table_name WHERE id=?,name=?";
PreparedStatement preparedStatement = conection.preparedStatement(sql);
//设置需要查询的信息，第一个参数为'?'的位置
preparedStatement.setInt(1,12);
preparedStatement.setString(2,"westmist");
//同样返回查找的结果集
ResultSet result = preparedStatement.execute();

Tip:需使用动态sql语句，必须设置数据库连接的useServerPrepStmts.

在conection的URL后加上?useServerPrepStmts=true这样才能保证mysql驱动会先把SQL语句发送给服务器进行预编译，然后在执行executeQuery()时只是把参数发送给服务器.
