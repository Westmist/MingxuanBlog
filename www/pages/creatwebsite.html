搭建Nginx静态网站
Nginx
Nginx
发布日期:   2019-07-28
作者:   明轩
阅读次数:  

一、准备阶段

  • 服务器，本文中使用的服务器为腾讯云服务器
  • SLL证书，本文使用DVSLL证书
  • Linux系统镜像Centos 7.6

二、域名的准备与解析

    1.在云控制台实例中添加解析记录

    2.使用ping命令验证是否解析成功

    3.在终端设备键入ping www.yourdomain.com如果返回显示你的域名则表明解析成功

注：www.yourdomain.com为你所申请的域名，域名的申请周期较长可早做准备。

三、安装Web服务

使用MobaXterm等软件采用SSH连接的方式远程登入服务器

  1. 使用yum命令安装nginx

    2. [root@VM_0_9_centos ~]# yum install -y nginx

  2. 启动nginx

    3. [root@VM_0_9_centos ~]# nginx

  3. 检查nginx是否安装成功，在浏览器输入你的域名出现nginx的测试界面则表明安装成功

  4. 设置nginx开机自启

    5. [root@VM_0_9_centos ~]# systemctl enable nginx.service

四、配置SSL证书

(1)、申请SSL证书

    申请过程中需要通过DNS验证，可选择解析验证或文件验证，解析验证需要在解析记录中加入一条指定值的TXT类型的解析记录。

(2)、上传安全证书和密钥文件

    DNS验证通过后，在SSL证书管理处下载文件，在解压后的文件夹中找到nginx文件夹，将其中的密钥文件使用WinSCP等文件传输软件上传到服务器的etc/nginx目录下。

(3)、更改nginx的配置文件

    路径为/etc/nginx/nginx.conf

    # HTTPS   ~   server
server {
listen 443     ssl http2 default_server;
listen     [::]:443 ssl http2 default_server;
#更改为你申请的域名
server_name www.yourdomain.com;
#更改为你的站点根目录 
root     /date/www;

#分别更改为你的证书和密钥的路径
ssl_certificate "/etc/nginx/1_www.yourdomain.com_bundle.crt";
ssl_certificate_key "/etc/nginx/2_www.yourdomain.com.key";
ssl_session_cache   shared:SSL:1m;
ssl_session_timeout   10m;
#请按照这个套件配置,配置加密套件,写法遵循 openssl 标准
ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
ssl_prefer_server_ciphers on;
(4)、nginx重定向

    nginx支持重定向功能,可以在nginx配置文件的80端口server块添加下行代码已实现HTTP自动跳转HTTPS的安全配置

    rewrite ^(.*)$ https://$host$1 permanent;

五、检查与验证

  1. 检查nginx配置文件是否生效，输入命令

    2. [root@VM_0_9_centos ~]# nginx -t

  2. 若配置文件无误，返回显示如下信息。如显示失败请检查文件的配置设置与语法

    3. nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful

  3. 重启nginx服务

    4. [root@VM_0_9_centos ~]# nginx -s reload

  4. 使用https协议访问你的域名，如https://www.yourdomain.com检查是否成功,如果地址栏出现锁头标志,说明SSL配置已成功
