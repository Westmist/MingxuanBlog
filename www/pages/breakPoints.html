响应式设计概述

该设计可使网站在不同的设备，上浏览时对应不同分辨率皆有适合的呈现，减少用户进行缩放、平移和滚动等操作行为.

断点设置 

    /*==================================================
    =            Bootstrap 3 Media Queries             =
    ==================================================*/
    
    /*==========  Mobile First Method  ==========*/
    
    /* Custom, iPhone Retina */ 
    @media only screen and (min-width : 320px) {
        
    }
    
    /* Extra Small Devices, Phones */

    }

相对单位

响应式设计一般不使用绝对单位px,而使用相对单位em或rem

    em:相对于元素font-size值的计量单位.

    如在元素样式中设置font-size:20px;则有1em = 20px;

    rem:相对于根元素的em(root element)

    该单位大小取决于<html>元素的font-size而不受元素字体尺寸的影响

    如设置<html style="font-size:10px">，则 1rem = 10px

设计原则

渐进增强（Progressive Enhancement）：一开始就针对低版本浏览器进行构建页面，完成基本的功能，然后再针对高级浏览器进行效果、交互、追加功能达到更好的体验.

优雅降级（Graceful Degradation）：一开始就构建站点的完整功能，然后针对浏览器测试和修复。比如一开始使用 CSS3 的特性构建了一个应用，然后逐步针对各大浏览器进行 hack 使其可以在低版本浏览器上正常浏览.

Tip：移动优先采取渐进增强原则，如主要是pc用户则可采用优雅降级的原则
